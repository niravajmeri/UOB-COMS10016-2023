COMS10016 | Week 01 | Preparation

SYSTEM SETUP FOR OSX

This is a brief tutorial on how to setup your personal Apple/Mac/OSX system to be ready for the COMS10016 unit. The below steps represent one out of many ways of setting up your system for the unit. Many of the steps can be shortened and they are outlined as is for clarity. Before you start make sure your system is up-to-date with respect to updates and features. It is your responsibility to keep your personal system and passwords safe. To ensure your Mac user has the required permissions, go to "System Preferences > Users & Groups". Ensure that your user account is listed as 'Admin', otherwise log in with an Admin account.

INSTALL THE CLANG COMPILER

  1. Open the Terminal (Mac command prompt). For example, press CMD + Space and type: Terminal then select Terminal.app - a Terminal window will open.
  2. Type: clang --version to see if the C compiler is already available.
    You should get ONE of the following responses:
    • Information shown such as 'Apple LLVM version 10.0.1 (...)'.
      The compiler is already installed, move on to Installing the VSCode Editor.
    • OR: A dialogue box pops up suggesting you install the Command Line Developer Tools.
      Select Install and follow the wizard steps to install these Tools.
      After this is complete, again type: clang --version
      You should now get output as above, move on to Installing the VSCode Editor.
    • OR: An error showing that the command is not found.
      Enter: xcode-select --install to start the Command Line Developer Tools install dialogue - follow instructions as above.
    If the above fails install 'Xcode12' and 'Command Line Tools for Xcode 12' from https://developer.apple.com/download/more.

INSTALL THE VSCODE EDITOR

Note that VSCode is the recommended text editor for this unit, but you can substitute it with your preferred editor/IDE if you wish.
  1. In your web browser, navigate to: https://code.visualstudio.com/docs/setup/mac
  2. Follow the 'Installation' and 'Launching from the command line' instructions

INSTALL HASKELL AND QUICKCHECK

  • Open the Terminal (Mac command prompt).
  • type "curl -sSL https://get.haskellstack.org/ | sh", then press enter and the update process will begin
  • Note: you will be asked for your root password for sudo purposes (approx. 2 times)
  • Note: after the installation, you may need to add "~/.local/bin" to your PATH variable. You can do this with the following terminal command: "PATH=$PATH:~/.local/bin"
  • Note: if you out of disk space, try deleting ".stack". This is a hidden file that you can see using "ls -a", and delete using "rm -fr .stack"
  • Note: if you have any further installation issues, try checking these FAQs. The Teaching Assistants (TAs) will be there in the first lab to help you with this in the setup labs.
  • after a successful installation, you can run the Haskell GHCi interpreter by typing "stack ghci" in the terminal and pressing enter
  • after a successful installation, you can run the Haskell GHCi interpreter with the QuickCheck package by typing "stack ghci --package QuickCheck" in the terminal and pressing enter
  • after a successful installation, you can interpret Haskell files (such as a file "File.hs") by adding the file as a final argument to your stack command like "stack ghci --package QuickCheck File.hs" in the terminal and pressing enter
  • after a successful installation, you can compile Haskell files (such as a file "File.hs") by adding the file as a final argument to your stack command like "stack ghc --package QuickCheck File.hs" in the terminal and pressing enter
  • Note: once you verify that GHCi is running, you can use the :q command to exit it and go back to the command prompt.
    • IMPORTANT NOTE: If the Haskell installation does not work for you then hold on and use the setup labs during week 1 to get help from a TA. Only if after asking TAs for help you have not been able to install Haskell successfully, then consider looking into more complicated alternatives.
At this point your system is ready for most of the COMS10016 unit and you are ready to start programming in C and Haskell. Further installations are optional at the beginning of the unit. However, during the last part of the unit we will use graphics, and for this we will need the SDL2 graphics libraries and a Linux desktop environment...

INSTALL THE SDL2 DEVELOPMENT PACKAGE

  1. Open the Terminal if not already open (see the first few steps above for how).
  2. The simplest (but not only) way to install SDL2 on a Mac is to use the HomeBrew package manager:
    • Type: brew to check if it's installed.
      If a 'command not found' error is shown, run the following command to install brew and follow the instructions (installation takes several minutes):
      /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
      • If you encounter a permissions issue on your device you may want to check out this discussion.
    • Type: brew update
    • To install SDL2, type: brew install sdl2
  3. Ensure SDL2 is installed by typing: sdl2-config --version (version information should be displayed, such as 2.0.10).

CHECK SDL2 GRAPHICS

  1. Download the program hellosdl.c to your current working directory (or change directory in the Terminal to the location of the downloaded file).
  2. Compile and run the program:
    • Use the following command to compile the program:
      clang -std=c11 -Wall -lSDL2 hellosdl.c -o hellosdl
      OR if this fails with a clang 'ld' linker error, try compiling with static (not dynamic) libraries using this command:
      clang -std=c11 -Wall `sdl2-config --static-libs` hellosdl.c -o hellosdl OR if this fails for M1s try: clang -std=c11 -Wall -o hellosdl hellosdl.c -lSDL2 -L/opt/homebrew/Cellar/sdl2/2.0.16/lib -I/opt/homebrew/Cellar/sdl2/2.0.16/include
    • If the compilation was sucessful, you will see the new executable file hellosdl in the current directory.
    • Run this program by typing: ./hellosdl
  3. If you see a sky blue 'HelloSDL' window appear then SDL2 is working properly.
    Setup is complete!
  4. Note that some ports of the SDL library unfortunately have memory leaks, which may be reported back to the terminal...you have to live with it.
